Open Source Platform for creating multi-device applications

Orchestrator.js is an open source platform for creating applications to heterogeneous and multi-device environment based on Node.js.
Make your devices to communicate seamlessly, and put their resouces into use!

Orchestrator.js

Orchestrator.js is a platform for new type of applications. The apps run constantly run in cloud and monitor device context as well as third party services, like Facebook, Flickr, etc. When rules set by the app developer are met, the app triggers interactions between devices and users. Users can start and stop different applications as they wish.

To have better controlr over their device, users can describe what their devices can or are allowed to do by enabling / disabling different device capabilities. For example, if device has a talking-capability enabled, it is then allowed to translate text to speech.

Orchestrator.js - Apps

Orchestrator.js applications are based on three main concepts.

App

The app itself monitors device context, and out-side sources like social media services, such as Facebook, Twitter and so on. Based on the observations and logic defined by the developer, the app triggers actions. So app defines when and what interactions (actions) should take place.

Action

An action defines how the devices interact with each other and as well as with users. An action can be very short, like just greeting each other, or long-lasting, like a game.

Capability

Capabilities are used to describe how a device appears to the system, what a device can or is allowed to do, and in what interactions the device can take part to. The capability can either be very generic, like Talking-capability that allows device to speak, or more specific, like Apocalympics-capability which allows the device to participate to Apocalympics game.

Orchestrator.js - Features

Building apps is easy

Orchestrator.js offers web-based IDE for implementing apps in heterogenous, multi-user and multi-device environment. The Web Console can be used for implementing the apps, interactions, and device capabilities. Moreover, the IDE offers tools for testing purposes: it is easy to monitor device states, read debug messages directly from web browser's console, and also developers can manually trigger interactions for a predefined set of devices.

Context is the key

With Orchestrator.js it is easy to implement context-aware applications for a heterogenous set of device. The platform offers publish-subsrcibe API that easy to utilize in app implementations. The devices constantly report their state to the Orchestrator.js Device Registry, which publishes updates whenever devices' contextual data changes.

Cloud-based system

The platform currently runs in Amazon cloud, but is freely available from GitHub, and can be easily hosted on any computer with a network connectivity. The devices are constantly connected to the Orchestrator.js. On client-side we currently support Android, and devices capable of running Python. We are also working on with iOS implementation currently.
